数据关联

现在已经可以输入姓名跟描述了，不过我们可能想偷懒下：“在姓名输入的时候，自动填写一个描述”。

怎么做呢？使用数据绑定这将很简单。

在 hero-new.form.plist > Item 1 > properties 添加 @input.text 绑定：

KeyTypeValue
@input.textString%!(Description of %@),=>name

这个表达式的几个细节：

  1. 使用了 =>name 表达式，=为单向绑定，>name对应属性域为 name 的输入控件(即姓名输入控件)的文本值
  2. 使用了 %! 格式化，这使得当 >name 为空时，描述也为空，而不会是 Description of 的奇怪样子

提交表单

现在我们来为表单加上一个按钮，以提交表单，创建 提交组件 submit.input.plist：

KeyTypeValue
RootDictionary(3 items)
viewsDictionary(1 item)
buttonDictionary(2 items)
clazzStringPBButton
propertiesDictionary(2 items)
clipsToBoundsBooleanYES
layer.cornerRadiusNumber5
metricsDictionary(2 items)
leftString12
topString24
formatsArray(2 items)
Item 0String|-left-[button]-left-|
Item 1StringV:|-top-[button]-top-|

这里使用了 PBButton 控件，该控件基于 UIButton，支持自动高亮、灰化背景色，并扩展支持了 title、action 等属性，细节参考PBButton控件。

修改 hero-new.form.plist，为 rows 添加一行元素：

KeyTypeValue
Item 2Dictionary(2 items)
layoutStringsubmit.input
propertiesDictionary(2 items)
@button.backgroundColorString#2baf2b
@button.titleStringSubmit

现在，我们需要在点击提交按钮时完成一系列交互行为：

  1. 验证并提交表单
  2. 发起 POST 请求
  3. 返回英雄列表页
  4. 添加新创建的英雄

对应的 action 为 submit -> post -> pop -> addRow。

我们为提交按钮的 properties 添加 action 属性：

KeyTypeValue
@button.actionDictionary(2 items)
typeStringsubmit
next.doneDictionary(3 items)
typeStringpost
nameStringheroes
next.doneDictionary(2 items)
typeStringpop
next.doneDictionary(1 item)
typeStringaddRow

可以看到 Action 流水线没有多余的数据设置，依赖 PBAction 的设计，自动完成了数据流的传递：

  1. submit 验证表单，验证通过后自动搜集输入控件的 <name, value> 键值对，保存至行为状态 state.params
  2. post 发起API请求，取 state.params 作为 POST 参数，请求成功后，保存数据至 state.data
  3. addRow 取 state.data 添加到当前列表视图

不过现在流水线还不能走通，我们需要为第2步模拟一个 POST 请求：

POST heroes

来返回我们刚刚添加的数据。

命令行进入工程目录，创建模拟JSON：

pbind mock PBClient heroes@post

修改 heroes@post.json：

{
  "$redirect": "#.params"
}

这里的关键字 $redirect 将重定向到指定数据作为返回， 而 #.paramsm 则映射到当前状态参数，即表单所提交的参数。
